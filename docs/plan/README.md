# docs/plan/ — Agent Planning Workflow

This directory holds planning documents generated by the Claude Code agent planning workflow. Plan files are `.md` documents that capture implementation scope, sequencing, and decision rationale during active development sessions.

---

## How It Works

Claude Code's planning tool creates structured markdown plan files that serve as living development artifacts:

1. **Decompose** — Break a goal into ordered steps with clear inputs, outputs, and verification criteria
2. **Track progress** — Each step carries a status (`not started` / `in progress` / `done`) updated in real time
3. **Recover context** — If a session crashes or restarts mid-build, the plan file lets the agent resume exactly where it left off (this happened during our Gen 1 build)
4. **Audit decisions** — The plan records *why* certain approaches were chosen (e.g., "raw LMP over forecast-compressed prices because $9k spikes dominate arbitrage revenue")

## Example: Gen 1 Build Session

The Gen 1 implementation used this workflow across two sessions:

```
Session 1: Plan → Build NB 01-03 → [crash mid NB-04]
Session 2: Read plan → Resume NB 04 → Audit vs Modo criteria → Add DA/RT + sensitivity → Docs → Push
```

The plan file enabled seamless recovery after the session crash, preserving all context about which sites had data, which notebooks were complete, and what remained.

Plan files are **development artifacts, not deliverables**. They are gitignored by default — only this README is tracked. The actual analytical methodology is documented in [`docs/solution/methodology.md`](../solution/methodology.md).
